# Advanced Tailwind CSS Techniques

Tailwind CSS has revolutionized how we write CSS, but there's much more to it than utility classes. Let's explore advanced techniques that will make you a Tailwind power user.

## Custom Configuration

Tailwind's power comes from its configurability. Let's customize it for your project:

```javascript

  theme: {
    extend: {
      colors: {
        primary: {
          50: '#eff6ff',
          500: '#3b82f6',
          900: '#1e3a8a',

        brand: {
          light: '#f8fafc',

        }

      fontFamily: {
        sans: ['Inter', 'system-ui', 'sans-serif'],
        mono: ['JetBrains Mono', 'monospace'],
      },

        'fade-in': 'fadeIn 0.5s ease-in-out',
        'slide-up': 'slideUp 0.3s ease-out',
      },
      keyframes: {
        fadeIn: {
          '0%': { opacity: '0' },
          '100%': { opacity: '1' },
```

## Component Architecture

Create reusable component patterns with Tailwind:

### Button Component

```jsx
const Button = ({ 
  variant = 'primary', 
  size = 'md', 
  children, 
  ...props 
}) => {
  const baseClasses = 'inline-flex items-center justify-center font-medium rounded-lg transition-colors focus:outline-none focus:ring-2 focus:ring-offset-2';
  
  const variants = {
    primary: 'bg-primary-500 text-white hover:bg-primary-600 focus:ring-primary-500',
    secondary: 'bg-gray-200 text-gray-900 hover:bg-gray-300 focus:ring-gray-500',
    danger: 'bg-red-500 text-white hover:bg-red-600 focus:ring-red-500',
  };
  
  const sizes = {
    sm: 'px-3 py-1.5 text-sm',
    md: 'px-4 py-2 text-base',
    lg: 'px-6 py-3 text-lg',
  };
  
  return (
    <button
      className={`${baseClasses} ${variants[variant]} ${sizes[size]}`}
      {...props}
    >
      {children}
    </button>
  );
};
```

### Card Component

```jsx
const Card = ({ children, className = '', ...props }) => {
  return (
    <div
      className={`
        bg-white rounded-lg shadow-sm border border-gray-200 
        hover:shadow-md transition-shadow duration-200
        ${className}
      `}
      {...props}
    >
      {children}
    </div>
  );
};

const CardHeader = ({ children, className = '' }) => (
  <div className={`px-6 py-4 border-b border-gray-200 ${className}`}>
    {children}
  </div>
);

const CardBody = ({ children, className = '' }) => (
  <div className={`px-6 py-4 ${className}`}>
    {children}
  </div>
);
```

## Responsive Design Patterns

### Mobile-First Approach

```jsx
// Mobile-first responsive design
<div className="
  w-full                    // Mobile: full width
  md:w-1/2                  // Medium: half width
  lg:w-1/3                  // Large: one-third width
  xl:w-1/4                  // Extra large: one-fourth width
">
  <div className="
    text-sm                  // Mobile: small text
    md:text-base             // Medium: base text
    lg:text-lg               // Large: large text
  ">
    Content here
  </div>
</div>
```

### Container Queries (Modern CSS)

```css
/* tailwind.config.js */
module.exports = {
  theme: {
    extend: {
      containerType: {
        'inline-size': 'inline-size',
      },
    },
  },
}
```

```jsx
<div className="container-type-inline-size">
  <div className="
    grid grid-cols-1 gap-4
    @container
    @md:grid-cols-2
    @lg:grid-cols-3
  ">
    {/* Cards that respond to container size */}
  </div>
</div>
```

## Best Practices

1. **Use semantic class names** with @apply for complex components
2. **Create design tokens** in your config for consistency
3. **Use responsive prefixes** consistently
4. **Optimize for performance** with proper purging
5. **Maintain accessibility** with focus states
6. **Document your patterns** for team consistency

Tailwind CSS is more than just utility classes. By mastering these advanced techniques, you can create maintainable, performant, and beautiful user interfaces. The key is to find the right balance between utility classes and component architecture for your specific project needs.
